## 🎯 Objetivo

Preparar tu espacio de trabajo y definir tu "protocolo de contexto" antes de interactuar con IA.

---

## ⚠️ Importante: No Copiar, Completar

**REGLA ORO**: Los ejercicios te hacen completar, no copiar. Esto te obliga a pensar.

---

## 1. Prerrequisitos

### 1.1. Software Necesario

- **Herramienta de IA**: ChatGPT, Claude, Gemini, Copilot, u otra de tu elección
- **Editor de notas**: Para guardar tu sistema de organización
- **Conexión a internet**: Para acceder a la IA

---

## 2. Escenario

Eres un developer que quiere usar IA de forma efectiva. Antes de escribir tu primer prompt serio, necesitas configurar tu approach.

---

## Pasos

### Paso 1: Elige tu Herramienta 🛠️

**Tu tarea**: Identifica la herramienta de IA que usarás durante este curso.

**Registro:**

```
Herramienta elegida: ___
```

---

### Paso 2: Define tu Sistema de Organización 📁

**Tu tarea**: ¿Cómo guardarás tus conversaciones?

Stark tenía una carpeta de "mantenimiento predictivo" para sus análisis de logs.

**Necesitas un sistema similar.**

**Piensa en:**
- ¿Carpetas por proyecto?
- ¿Tags para categorizar?
- ¿Un sistema de naming?
- ¿Notas junto a cada conversación?

**Registro:**

```
Mi sistema de organización:
1. ___
2. ___
3. ___
```

---

### Paso 3: Tu Contexto Default 📋

**Tu tarea**: ¿Qué información siempre incluyes en tus prompts?

Stark siempre incluía:
- El dominio/proyecto
- El formato esperado
- Los constraints importantes
- Las limitaciones conocidas

**Tu lista de contexto default:**

```
Siempre incluyo:
1. ___
2. ___
3. ___
```

---

### Paso 4: Tu Primer Prompt (Simulado) ✍️

**Tu tarea**: Imagina que necesitas crear una función que valide passwords.

**Versión SIN contexto:**
```
Escríbeme una función que valide passwords
```

**Versión CON contexto (completa esta):**
```
Escríbeme una función que valide passwords para ___
Los passwords deben cumplir: ___
Formato de salida: ___
Manejar estos casos especiales: ___
```
